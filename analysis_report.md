# Sales Data Analysis Report

## Project Overview
This project performs an exploratory analysis of a sales dataset using Python and the Pandas library. The main objective is to understand sales performance by analyzing products, quantities, prices, and regions. Various statistical operations and aggregation techniques were used to extract meaningful insights from the dataset.

# Dataset Information
The dataset contains sales transaction records with the following columns:
* Customer_ID
* Product
* Category
* Region
* Quantity
* Price
* Total_Sales
The dataset was imported using the Pandas library and examined before performing any analysis.

# Data Exploration
The following exploratory steps were carried out:
* Loaded the dataset using `pd.read_csv()`.
* Displayed the first few records using `head()`.
* Checked the shape of the dataset to determine the number of rows and columns.
* Retrieved the list of column names.
* Generated descriptive statistics using `describe()`.
* 
# Data Cleaning
The dataset was inspected for missing values using:
```
dataset.isnull().sum()
```
Result:
* No missing values were found in any column.
* Therefore, no data cleaning or imputation was required.
An additional column named **total_sales** was temporarily created by multiplying **Quantity** and **Price** for comparison purposes. This temporary column was later removed to keep the dataset clean.

# Analysis Performed
## 1. Average Sales by Product
Calculated the average value of **Total_Sales** for each product using the `groupby()` function.
Purpose:
* To understand the average revenue generated by each product.


## 2. Number of Products Sold by Region
Calculated the total number of products sold in every region.
Purpose:
* To compare sales activity across different regions.

## 3. Total Quantity Sold by Region
Calculated the total quantity of products sold in each region.
Purpose:
* To identify regions with higher sales volume.

## 4. Highest Product Price
Found the maximum selling price for every product.
Purpose:
* To identify the highest recorded price of each product.

## 5. Lowest Product Price
Calculated the minimum selling price for each product.
Purpose:
* To observe the lowest price at which each product was sold.

## 6. Average Product Price
Calculated the average selling price for each product.
Purpose:
* To understand the average pricing of products.

## 7. Product Count
Counted the number of records available for every product.
Purpose:
* To determine how frequently each product appears in the dataset.

## 8. Maximum Sales by Product
Calculated the highest **Total_Sales** value recorded for each product.
The results were further sorted in descending order to identify the top-performing products.

## 9. Total Quantity Sold by Product
Calculated the cumulative quantity sold for each product.
Purpose:
* To identify products with the highest sales volume.

## 10. Overall Sales Metrics
The following overall business metrics were calculated:
* Total Sales Revenue
* Total Number of Customers
* Total Quantity Sold
These metrics provide a high-level summary of the overall sales performance.
---
## 11. Distinct Products
Retrieved all unique product names using the `unique()` function.
Purpose:
* To identify the different products available in the dataset.
